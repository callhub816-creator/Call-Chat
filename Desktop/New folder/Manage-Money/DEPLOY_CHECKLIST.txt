PHASE 1: PRE-DEPLOYMENT VERIFICATION
================================================================================

FUNCTIONAL TESTING:
[ ] npm install runs without errors

[ ] Mobile responsiveness tested (iPhone, Android, tablet)
[ ] No console errors in browser DevTools

[ ] Sitemap.xml accessible at /public/sitemap.xml
[ ] Robots.txt accessible and proper format
[ ] HTTPS enabled (add SSL before deploying live)
[ ] Favicon displays correctly
[ ] Images load and have alt text
[ ] Internal links work correctly
[ ] External links open in new tabs where appropriate

COMPLIANCE & SECURITY:
[ ] Privacy Policy page accessible
[ ] Disclaimer page accessible
[ ] About page has contact email
[ ] No API keys or secrets in code (.env.example only)
[ ] No console.log() debugging statements left in code
[ ] .gitignore includes .env and node_modules
[ ] No hardcoded localhost URLs (all relative paths)

PERFORMANCE:
[ ] PageSpeed Insights score >60 (desktop and mobile)
[ ] Lighthouse accessibility score >85
[ ] Lighthouse best practices score >85
[ ] Time to First Contentful Paint < 2.5s
[ ] No 404 errors in network tab
[ ] Images optimized (< 100KB each)

CONTENT REVIEW:
[ ] All guides have proper disclaimers
[ ] No spelling or grammar errors in visible content
[ ] Sample article includes sources and FAQs
[ ] Footer has links to legal pages

PHASE 4: LAUNCH DAY STEPS
================================================================================

HOUR-BY-HOUR:

2 HOURS BEFORE LAUNCH:
[ ] Final build test: npm run build && npm run preview
[ ] DNS cache purge (if using custom domain)
[ ] Notify team/stakeholders
[ ] Have rollback plan ready

1 HOUR BEFORE LAUNCH:
[ ] Deployment environment prepared
[ ] All environment variables confirmed
[ ] Deployment scripts tested
[ ] Log monitoring tools ready

AT LAUNCH TIME:
[ ] Deploy to production environment
[ ] Verify deployment successful (check provider's dashboard)
[ ] Test live domain in multiple browsers
[ ] Verify all pages load
[ ] Check for errors in browser console
[ ] Monitor server logs for errors

30 MINUTES AFTER LAUNCH:
[ ] Performance baseline recorded (PageSpeed, Core Web Vitals)
[ ] Monitor error rates and exceptions
[ ] Check Analytics data coming through
[ ] Spot-check key pages
[ ] Verify contact form working

2 HOURS AFTER LAUNCH:
[ ] Traffic patterns normal (check Analytics)
[ ] No repeated errors in logs
[ ] CDN cache status healthy (if using CDN)
[ ] DNS propagation confirmed (nslookup check)
[ ] Send success notification to team

================================================================================
PHASE 5: POST-LAUNCH (First Week)
================================================================================

DAY 1:
[ ] Monitor error logs (no critical issues)
[ ] Analytics tracking confirmed (GA4 recording visits)
[ ] PageSpeed monitored (no degradation)
[ ] User feedback collected (Contact form checked)

DAY 2-3:
[ ] Submit sitemap to Search Console (if not auto-discovered)
[ ] Request URL indexing in Search Console for key pages
[ ] Monitor AdSense readiness (if planning to apply)
[ ] Content quality spot-check (all pages render correctly)

DAY 4-7:
[ ] Google Search Console: Monitor for crawl errors
[ ] Analytics: Verify traffic patterns and sources
[ ] Performance: Check Lighthouse scores remain >85
[ ] Content: Verify all articles indexed in Google
[ ] Preparation: Begin AdSense application process

================================================================================
PHASE 6: ADSENSE APPLICATION (Week 2)
================================================================================

BEFORE APPLYING:
[ ] 30+ days of Google Analytics data available
[ ] 15+ high-quality articles published and indexed
[ ] Privacy Policy complete and accessible
[ ] Contact page with working email
[ ] All legal pages (About, Disclaimer, Privacy) complete
[ ] No policy violations detected
[ ] PageSpeed Insights score >60 on all pages

APPLY FOR ADSENSE:
[ ] Go to https://www.google.com/adsense/
[ ] Sign in with Google account
[ ] Click "Sign up" and enter domain
[ ] Wait for initial approval (24-48 hours typically)
[ ] Check email for approval or denial

IF APPROVED:
[ ] Log into AdSense dashboard
[ ] Generate ad code for placement
[ ] Create ad units (728x90, 300x250, 336x280 recommended)
[ ] Implement ad code in Layout.tsx (replace comments)
[ ] Test ads with AdSense preview mode (do not click real ads)
[ ] Monitor earnings for first month

IF DENIED:
[ ] Read Google's denial explanation carefully
[ ] Fix identified issues (typically: content quality, traffic, compliance)
[ ] Wait 2 weeks minimum before reapplying
[ ] Reapply with improvements
[ ] Document each attempt and fixes made
